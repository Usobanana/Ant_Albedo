# プロジェクト「Ant_Albedo」開発ロードマップ

## プロジェクト概要
- **ゴール**: 小規模ゲームのリリースとAI開発ナレッジの体系化。
- **主要な役割**:
    - **Project Manager (PM)**: 進行管理、フォルダ構成、ドキュメント管理。
    - **Lead Engineer (LE)**: 技術選定、実装、アーキテクチャ設計。
    - **Product Manager (PdM)**: 各フェーズの承認、品質評価、俯瞰的視点。

## フェーズ別進捗管理

| フェーズ | 状態 | 内容 | 担当役割 |
| :--- | :--- | :--- | :--- |
| ①企画立案 | 完了 | ゲーム概要、コアサイクルの言語化 | PM/PdM |
| ②仕様策定 | 完了 | 画面遷移、操作、ルールをmd化 | PM/LE |
| ③要件/実装策定 | 完了 | 技術スタック、データ構造、実装方針 | LE |
| ④開発承認 | 完了 | PdMによる俯瞰的レビューと承認 | PdM |
| ⑤プロトタイプ | 完了 | コア機能の実装とテスト | LE |
| ⑥プロト承認 | 完了 | 仕様と実装の整合性確認 | PdM |
| ⑦アルファー | 進行中 | 主要サイクルの構築・ディレクター調整 | LE/Designer |
| ⑧アルファー承認 | 未着手 | **ディレクターによる最終承認** | Director |
| ⑨ベータ | 未着手 | 全機能の実装・磨き込み | LE |

## 🚩 マイルストーン・ガバナンス
各開発フェーズの完了（AlphaからBetaへの移行等）は、**ディレクター（USER）の明示的な承認** があった場合のみ確定します。

## AI開発ナレッジ管理 (Update: 2026-04-26)
- **[P3] ディレクター主導の現場調整**:
    - AIに「ゲームデザイナー」の役割を持たせ、デバッグメニューを介してディレクターと対話させることで、感覚的な調整を迅速に言語化・数値化できる。
- **[P4] 負のフィードバック・ループの防止**:
    - 時間経過による難易度上昇ロジックにおいて、計算式が負の値にならないよう `Math.max` 等でガードを徹底することの重要性（無限増殖バグの教訓）。
- **[P5] 入力閾値の遊び**:
    - 「短押し」と「ドラッグ」の判定は、理想的な数値（10px）よりも余裕を持たせた数値（40px）の方が、ユーザー体験の安定に繋がる。
