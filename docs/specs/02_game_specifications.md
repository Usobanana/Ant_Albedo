# ゲーム仕様・実装設計書 (Phase 2 & 3)

## 1. 画面構成 (Layout)
- **全体画面**: 縦長 (Mobile First)
- **上部 (Battle Area)**:
    - 横スクロールのバトルフィールド。
    - 左端：プレイヤー拠点 (HPゲージ)
    - 右端：敵拠点/出現ポイント
- **下部 (Merge Area)**:
    - 5x5 のグリッド。
    - アイテム（素材・モンスター）が配置される。

## 2. マージシステムの詳細
- **アイテム出現**:
    - 3秒ごとに、**「選択したデッキ」に含まれるLv0アイテム**がランダムに空きマスに1つ出現。
- **合成ルール**:
    - **同じ種類 かつ 同じレベル** のアイテムを重ねると進化。
    - 種類が違う場合は合成できない。
- **出撃 (Deploy)**:
    - Lv1 以上のユニットをバトルエリアへスワイプで送り出す。
    - ユニットの種類に応じた特性（近接、遠距離など）が戦場で反映される。

## 3. バトルシステムの詳細
- **ユニットの多様性**:
    - 種類ごとに「攻撃力」「射程」「速度」「HP」が異なる。
- **敵の挙動**:
    - ステージごとに決まったパターンで出現。
- **ステージプレビュー**:
    - プレイ開始前に「出現する敵の情報」が表示され、プレイヤーはそれを見てデッキを編成する。

## 4. アルファー開発での追加要素 (Alpha Scope)
- **デッキ編成画面**: 全5種類以上のユニットから、持ち込む3〜4種類を選択。
- **ユニット図鑑**: 各ユニットの進化ツリーと能力を確認できる。
- **複数ステージ実装**: 徐々に難易度が上がるステージ構成。

## 4. 機能要件 (Technical Requirements)
- **状態管理 (State)**:
    - `gridState`: マスごとのアイテムLvを保持する配列。
    - `units`: 戦場にいる味方ユニットのインスタンスリスト。
    - `enemies`: 戦場にいる敵ユニットのインスタンスリスト。
- **主要コンポーネント**:
    - `GameEngine`: 全体のループ（更新、描画、衝突判定）を管理。
    - `MergeManager`: ドラッグ＆ドロップと合成判定を担当。
    - `BattleManager`: ユニットの移動、攻撃、HP管理を担当。

## 5. AI開発ナレッジ検証項目
- 「ドラッグ＆ドロップという複雑なUI操作のコードを、AIがどこまで正確に書けるか」
- 「バトルとパズルの異なる時間軸（Tick）をどう同期させるか」
- 「CSSによるアニメーションとJSのロジックの分離」
