# バトルシステム仕様 (1ライン・カメラ追従版)

## 1. 戦場の構造
- **1ライン制**: 全ユニットは単一の進軍路上で戦う。
- **奥行き表現**: 見た目上の重なりを避けるため、Y軸方向に微細なランダムオフセット（奥行き）を設ける。
- **広大なマップ**: ステージごとに横幅（px）が定義され、画面外へスクロール可能。

## 2. キャラクター
- **勇者 (Hero)**: プレイヤー側の拠点。マップ左端に位置。
- **魔王 (Demon King)**: エネミー側の拠点。マップ右端に位置。
- **ユニット/敵**: 勇者から出撃し、魔王を目指す。

## 3. カメラ制御 (Smart Camera)
- **初期表示**: 勇者を画面中央付近に見せる状態から開始。
- **自動追従 (Auto-Follow)**: 味方ユニットの「最前線」を追従し、常に戦況のメインを画面に収める。
- **マニュアル操作**: ユーザーによるスワイプ中は追従を無効化。操作停止後、前線が視界内にあれば追従を再開する。

## 4. 戦闘ロジック
- **接触判定**: X軸上の距離で判定。
- **スタック**: 先頭のユニットが敵とぶつかると、後続もその位置で足を止め、戦闘に参加する（擬似的な戦列）。

## 5. 配置（デプロイ）
- **自陣（配置可能エリア）**:
    - 拠点（x=150）から、現在戦場にいる味方ユニットの「最前線（最も大きいX座標）」までの範囲。
    - 味方ユニットがいない場合は、拠点付近のみが対象。
- **召喚ルール**:
    - 自陣エリア内であれば、ドラッグ＆ドロップした指のX座標に直接ユニットを配置できる。
    - 自陣エリア外（敵陣寄り）にドロップした場合は、自動的に「現在の最前線」に補正されて配置される。
    - これにより、マージしてから前線に到達するまでのタイムラグを戦略的に短縮可能。
