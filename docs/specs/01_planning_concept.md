# 企画立案 (Phase 1) - ゲームコンセプト案

## 1. ゲームタイトル（仮）
- **Merge & Defense (仮)**

## 2. ゲームの概要
- **ジャンル**: マージパズル × ラインディフェンス
- **プラットフォーム**: Web (PC/Mobile) ※プロトタイプはブラウザベースを想定
- **ターゲット層**: 隙間時間に遊びたいカジュアルゲーマー、育成・合成が好きな層

## 3. コア体験 (UX)
- **「合成の快感」**: 同じアイテムをくっつけて強化する手軽な楽しさ。
- **「援護の達成感」**: 自分が育てたキャラを戦場に送り出し、敵を押し返すカタルシス。

## 4. コアサイクル (Game Loop)
1. **[Merge]**: 下画面のグリッドに時間経過で出現する素材を合成し、ユニット（モンスター）を生成・強化する。
2. **[Deploy]**: 生成したユニットを上画面のバトルフィールドにスワイプ（またはドロップ）で送り出す。
3. **[Battle]**: 送り出されたユニットが自動で右へ進軍し、迫りくる敵と戦う。
4. **[Result]**: 敵の拠点（または全滅）を達成すれば勝利。リソースを得て、さらなる強化へ。

## 5. 特徴・ユニークポイント
- パズルとバトルの同時並行による、ほどよい忙しさ。
- AIを活用した「モンスターの進化系統」や「敵の出現パターン」の生成。

## 6. 使用技術（Lead Engineer 案）
- **開発言語**: HTML5 / JavaScript (Vanilla または Viteベース)
- **理由**: 実行環境の構築が容易で、AIにコードを生成させやすく、ブラウザですぐに確認できるため。
- **グラフィック**: AIツール（DALL-E / Midjourney等）で生成した素材、またはシンプルなシェイプ。
